# Realmforge (铸造世界) — Project CLAUDE.md

## 自动提交

每次完成有意义的变更后，自动 git commit and push：

```bash
git add -A && git commit -s -m "<type>: <description>" && git push
```

- 使用约定式提交前缀：`feat` / `fix` / `docs` / `refactor` / `test` / `chore`
- 提交信息用中文，简洁描述变更内容
- push 到当前分支的远端

## 文档目录结构

### 原文与转化分离原则

```
docs/
├── 基础资料/          # 原始资料原文（不可直接用于游戏设计）
│   ├── 山海经/        #   山海经各篇原文整理
│   ├── 本草纲目/      #   本草纲目原文
│   └── 山海经_世界总览.md
│
├── 动物/              # 原材料 — 动物类（可用于游戏设计）
│   ├── 山海经/        #   山海经动物 → 瑞兽/凶兽/食人兽/异兽/鸟类/蛇类/海中生物
│   └── 本草纲目/      #   本草纲目动物（待整理）
├── 植物/              # 原材料 — 植物类
│   ├── 山海经/        #   山海经植物 → 神奇树木/神奇草本
│   └── 本草纲目/      #   本草纲目植物（待整理）
├── 矿石/              # 原材料 — 矿石宝物类
│   ├── 山海经/        #   山海经矿石 → 玉石/金属/特殊矿物
│   └── 本草纲目/      #   本草纲目矿石（待整理）
│
├── 药水/              # 制品 — 药水类（依赖动物/植物/矿石原材料）
├── 丹药/              # 制品 — 丹药类（依赖动物/植物/矿石原材料）
└── 武器/              # 制品 — 武器类（依赖动物/植物/矿石原材料）
```

**核心规则**：
- `基础资料/` 存放 **原文**，保持原始文献的完整性
- `动物/`、`植物/`、`矿石/` 存放 **原材料** — 从基础资料转化的结构化数据
- `药水/`、`丹药/`、`武器/` 存放 **制品** — 依赖原材料组合生成的内容
- 所有资料源（山海经、本草纲目、后续其他资料）按类别归入对应目录
- 转化文件需标注来源，如 `> 来源：[山海经_世界总览.md](../基础资料/山海经_世界总览.md)`

**依赖关系**：
```
基础资料（山海经、本草纲目等）
    ↓ 提取
原材料：动物 / 植物 / 矿石
    ↓ 组合
制品：药水 / 丹药 / 武器
```

## 统一编码体系

所有资料条目使用统一编码，建立跨文档关联。

| 大类 | 子类 | 编码前缀 | 说明 |
|------|------|----------|------|

| 地理·水 | — | `水-` | 河流、渊、泽、特殊水域 |

| 地理·方 | — | `方-` | 四海与方位 |

| 兽·食人 | — | `兽食-` | 食人之异兽 |

| 鸟 | — | `鸟-` | 鸟类 |

### 使用方式

- 每个条目在名称后标注编码，如 `**凤凰** [兽瑞-001]`
- 跨文档引用时直接使用编码，如 `见 [兽瑞-001] 则天下安宁`

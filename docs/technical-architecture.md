## 1. 产品概述

### 1.1 核心玩法循环

```
现实世界拍照 ──→ AI识别材料 ──→ 锻造装备/道具
     │
儿童绘画 ──→ AI美化 ──→ 属性计算 ──→ 锻造/培育山海经异兽
     │
异兽战斗/探索 ──→ 发现新材料 ──→ 异兽变异/进化
     │
世界探索 ──→ 贡献内容到共享世界 ──→ 与好友互动
```

### 1.2 核心系统

| 系统 | 说明 |
|------|------|
| **材料识别系统** | 拍摄现实物品，AI识别并转化为游戏内锻造材料 |
| **绘画锻造系统** | 儿童绘制设计图，AI美化后计算属性并锻造 |
| **异兽培育系统** | 基于《山海经》生成异兽，支持基因变异与进化 |
| **共享世界系统** | 所有玩家的贡献汇聚成持久化的共享山海经世界 |
| **阵营对抗系统** | 两大阵营（如"铸天"与"炼地"）的竞争与合作 |

## 2. 技术选型

### 2.1 游戏引擎

| 维度 | **Godot 4.x**（推荐） | Unity | Web (Phaser/Three.js) |
|------|----------------------|-------|----------------------|
| 许可 | MIT 开源，无版税 | 免费但有营收阈值 | 开源 |
| 2D 支持 | 优秀（节点树天然适合2D） | 良好 | 依赖库 |
| 3D 支持 | 良好（Vulkan渲染） | 优秀 | Three.js 成熟 |
| 跨平台 | 导出 Android/iOS/Web/Desktop | 全平台 | 浏览器 |
| 包体大小 | 小（~40MB 空项目） | 较大 | 取决于资源 |
| 学习曲线 | 平缓（GDScript类Python） | 中等 | 中等 |
| AI SDK 集成 | HTTP 调用后端 API | 丰富插件生态 | 原生 JS/TS |
| 儿童友好 | 轻量、可定制 UI | 成熟但较重 | 无需安装 |

**决策：选择 Godot 4.3+**

理由：
- 开源无版税，对儿童产品无商业风险
- GDScript 类 Python 语法，团队学习成本低
- 包体小，儿童设备（中低端平板/手机）友好
- 2D 节点系统非常适合《山海经》风格的横版/俯视角表现
- 可后续通过 GDExtension 接入 C++/Rust 高性能模块

### 2.2 后端技术栈

```
┌─────────────────────────────────────────────────────────┐
│                    后端技术栈                            │
├──────────────┬──────────────────────────────────────────┤
│ API 框架     │ FastAPI (Python)                          │
│ AI 服务      │ 阿里云百炼 / OpenAI API / 本地部署模型     │
│ 数据库       │ PostgreSQL (主数据) + Redis (缓存/会话)    │
│ 对象存储     │ 阿里云 OSS / AWS S3 (图片/绘画文件)        │
│ 消息队列     │ Redis Streams / RabbitMQ (异步任务)        │
│ WebSocket    │ FastAPI WebSocket (实时交互)              │
│ 内容审核     │ 阿里云内容安全 API                        │
│ 部署         │ Docker + K8s / 阿里云 ACK                 │
└──────────────┴──────────────────────────────────────────┘
```

**选择 Python 作为后端语言的理由：**
- AI/ML 生态最成熟（PyTorch, OpenCV, transformers）
- 与 `uv` 工具链完美集成（依赖管理、打包）
- 快速迭代，适合原型验证
- 团队若有游戏逻辑脚本需求，与 GDScript 思维一致

### 2.3 AI/ML 技术选型

| AI 能力 | 方案 | 说明 |
|---------|------|------|
| **图像识别** | 阿里云视觉智能 / CLIP + 自训练分类器 | 识别现实物品类型、材质、颜色 |
| **绘画美化** | Stable Diffusion + ControlNet | 保留儿童笔触意图，提升画面质量 |
| **属性计算** | 自研规则引擎 + 轻量 ML 模型 | 基于物理属性推算游戏数值 |
| **异兽生成** | SD + LoRA (山海经风格微调) | 生成符合山海经风格的异兽图像 |
| **内容审核** | 阿里云内容安全 / Azure Content Safety | 自动过滤不当图片 |
| **文本生成** | 通义千问 / Claude API | 生成异兽描述、故事背景 |

### 2.4 目标平台优先级

```
Phase 1: 平板 (iPad / Android Pad)  ← 儿童主要使用场景
Phase 2: 手机 (iOS / Android)
Phase 3: Web (浏览器，降低门槛)
Phase 4: PC/Mac (家庭场景)
```

**平板优先的理由：**
- 绘画体验最佳（大屏 + 触控笔）
- 拍照操作对儿童更友好
- 家长更容易监督使用情况
- 屏幕空间足够展示复杂 UI

#### 3.2.3 异兽生成与变异

```
异兽生成流程：
  1. 选择/获得基础山海经异兽模板（如九尾狐、毕方、饕餮）
  2. 投入锻造材料作为"基因原料"
  3. AI 属性计算引擎基于材料组合推算异兽属性
  4. SD + LoRA 生成异兽视觉形象（保留山海经风格）
  5. 生成唯一基因编码

异兽变异流程：
  1. 两只异兽"融合"或异兽+特殊材料
  2. 基因编码交叉 + 随机突变
  3. 新属性 = f(父代属性, 材料属性, 随机因子)
  4. 生成新异兽形象（视觉上有父母特征 + 变异特征）
  5. 如果足够独特，可提交到共享世界
```

**世界区块设计：**
- 将《山海经》的地理划分为网格区块（Grid-based）

- 区块状态定时聚合更新，非实时（降低服务器压力）

## 4. 数据架构

### 4.1 核心数据模型

#### 4.1.1 材料 (Material)

```
Material
├── id: UUID (主键)

├── name: string (如 "玄铁矿", "扶桑叶")
├── rarity: enum [common, uncommon, rare, epic, legendary]
├── attributes: JSONB
│   ├── hardness: int (1-10)
│   ├── weight: enum [light, medium, heavy]
│   ├── element: enum [metal, wood, water, fire, earth, wind, lightning, dark, light]
│   ├── shanhai_origin: string (如 "南山经", "海外东经")
│   └── magic_affinity: int (1-10)
├── discovered_by: UUID (玩家ID)
├── source_image_url: string (原始照片URL)
├── created_at: timestamp
└── world_contribution: bool (是否已贡献到共享世界)
```

#### 4.1.2 装备/道具 (Equipment)

```
Equipment
├── id: UUID
├── owner_id: UUID
├── name: string
├── type: enum [weapon, armor, tool, accessory, consumable]
├── image_url: string (AI美化后的图片)
├── original_drawing_url: string (儿童原始绘画)
├── attributes: JSONB
│   ├── attack: int
│   ├── defense: int
│   ├── speed: int
│   ├── element: enum
│   ├── special_effect: string
│   └── durability: int
├── material_ids: UUID[] (使用的材料列表)
├── forge_recipe_hash: string (锻造配方哈希，用于复现)
├── created_at: timestamp
└── world_shared: bool
```

#### 4.1.3 异兽基因编码 (Beast)

**这是系统的核心设计——如何保证异兽的唯一性和可遗传性。**

```
Beast
├── id: UUID
├── name: string (可自定义)
├── species_id: UUID (基础物种模板，如 "九尾狐")
├── owner_id: UUID
├── gene_encoding: JSONB
│   ├── version: int (基因编码版本)
│   ├── genome: string (Base64 编码的基因序列，如 "A3F8B2C1D9E4...")
│   │   // 基因组结构：
│   │   // 每4位一组代表一个基因性状
│   │   // [形态基因][颜色基因][属性基因][特殊基因][稀有度基因][变异标记]
│   ├── traits: {
│   │   ├── body_type: int      // 体型
│   │   ├── color_primary: int  // 主色
│   │   ├── color_secondary: int // 副色
│   │   ├── element_affinity: int // 元素亲和力
│   │   ├── base_stats: {       // 基础属性
│   │   │   ├── strength: int
│   │   │   ├── agility: int
│   │   │   ├── vitality: int
│   │   │   └── spirit: int
│   │   │   }
│   │   ├── special_abilities: string[] // 特殊能力列表
│   │   └── mutation_markers: string[]  // 变异标记
│   │   }
│   └── generation: int (第几代)
├── image_url: string (AI生成图像)
├── parent_ids: UUID[] (父母ID，可为空)
├── mutation_type: enum [none, natural, material_forced, world_event]
├── personality: JSONB (性格，影响行为表现)
├── level: int
├── experience: int
├── created_at: timestamp
└── world_legend: bool (是否成为世界传说异兽)
```

**基因编码设计说明：**

```
基因组示例: "A3F8-B2C1-D9E4-F1A7-B8C3-D2E5"

基因段解析：
  A3F8 → 形态基因 (决定体型、角、翼、尾等外形特征)
  B2C1 → 颜色基因 (决定毛色、纹理、光效)
  D9E4 → 属性基因 (决定元素属性、基础数值倾向)
  F1A7 → 特殊基因 (决定特殊能力、天赋)
  B8C3 → 稀有度基因 (影响整体稀有度权重)
  D2E5 → 变异标记 (标记是否有特殊变异，用于遗传)

遗传规则：
  - 子代基因组 = 父代1(50%) + 父代2(50%) + 随机突变(0-5位)
  - 材料强制变异：投入特殊材料可定向改变特定基因段
  - 世界事件变异：极低概率触发稀有变异
```

# 伪代码：绘画美化服务
async def beautify_drawing(
    drawing_image: Image,
    intent_hint: str | None,
    beautify_level: float = 0.5,
    style: str = "shanhai"
) -> BeautifyResult:
    # 1. 意图理解
    if intent_hint is None:
        intent_hint = await clip_predict_intent(drawing_image)

    # 2. ControlNet Scribble 预处理
    scribble_map = extract_scribble(drawing_image)

    # 3. 生成美化图
    beautified = await stable_diffusion.generate(
        prompt=f"{style} style, {intent_hint}, masterpiece, best quality",
        negative_prompt="realistic, photo, ugly, deformed",
        controlnet_input=scribble_map,
        controlnet_strength=beautify_level,
        lora="shanhai_beasts_v2",
        seed=random_seed()
    )

    # 4. 意图保留验证
    similarity = compute_ssim(drawing_image, beautified)
    if similarity < 0.4:
        # 重生成或降低强度
        beautify_level *= 0.8
        beautified = await retry_generate(...)

    # 5. 内容安全检查
    await content_safety.check(beautified)

    return BeautifyResult(
        image=beautified,
        intent=intent_hint,
        similarity=similarity,
        original_preserved=True
    )

### 5.3 属性计算引擎

**核心：将绘画特征 + 材料属性映射为游戏数值。**

```
┌─────────────────────────────────────────────────────────────┐
│                    属性计算引擎                              │
│                                                             │
│  输入: 绘画特征 + 材料属性 + 锻造配方                        │
│  输出: 装备/异兽属性 (攻击/防御/速度/元素/特效)              │
│                                                             │
│  计算模块:                                                   │
│                                                             │
│  1. 形态分析 (Shape Analysis)                                │
│     ├── 轮廓复杂度 → 基础属性总量                            │
│     ├── 尖锐度 → 攻击倾向                                   │
│     ├── 圆润度 → 防御倾向                                   │
│     ├── 对称性 → 稳定性/速度                                │
│     └── 面积占比 → 威力大小                                 │
│                                                             │
│  2. 颜色分析 (Color Analysis)                                │
│     ├── 主色 → 元素属性                                     │
│     │   (红→火, 蓝→水, 绿→木, 黄→土, 白→金, 黑→暗, 紫→雷电) │
│     ├── 色彩丰富度 → 属性多样性                             │
│     └── 饱和度 → 属性强度                                   │
│                                                             │
│  3. 材料加权 (Material Weighting)                            │
│     ├── 材料基础属性 × 数量                                  │
│     ├── 材料组合加成 (相生相克)                              │
│     │   (五行相生: 木→火→土→金→水→木)                       │
│     │   (五行相克: 木→土→水→火→金→木)                       │
│     └── 稀有材料倍率                                        │
│                                                             │
│  4. 配方加成 (Recipe Bonus)                                  │
│     ├── 已知配方: 额外加成                                   │
│     ├── 创新配方: 随机特殊属性                               │
│     └── 五行平衡度 → 品质评级                               │
│                                                             │
│  5. 随机种子 (Seed Factor)                                   │
│     ├── 基于时间戳 + 玩家ID + 哈希                           │
│     └── 确保相同输入产生相同输出 (可复现性)                  │
└─────────────────────────────────────────────────────────────┘
```

**属性计算公式概念：**

```python
# 伪代码：属性计算
def calculate_attributes(drawing_features, materials, recipe):
    # 形态因子
    shape_scores = analyze_shape(drawing_features)
    # 颜色因子
    color_scores = analyze_colors(drawing_features)
    # 材料因子
    material_scores = aggregate_material_attributes(materials)

    # 五行计算
    wuxing = compute_wuxing_balance(material_scores, color_scores)

    # 基础属性
    base = {
        'attack': shape_scores['sharpness'] * 10 + material_scores['hardness'] * 5,
        'defense': shape_scores['roundness'] * 10 + material_scores['density'] * 5,
        'speed': shape_scores['symmetry'] * 8 + material_scores['weight_factor'] * 3,
        'spirit': color_scores['saturation'] * 6 + wuxing['balance_bonus'] * 10,
    }

    # 元素属性
    element = determine_primary_element(color_scores, material_scores)

    # 五行加成
    if wuxing['is_balanced']:
        for stat in base:
            base[stat] *= 1.2  # 五行平衡加成

    # 材料组合特效
    special_effects = check_material_combinations(materials)

    # 随机种子确保可复现
    seed = hash(recipe + player_id + timestamp)
    rng = SeededRNG(seed)
    base = apply_random_variance(base, rng, variance=0.1)

    return EquipmentAttributes(**base, element=element, special_effects=special_effects)
```

### 7.2 隐私保护

```
隐私设计原则 (COPPA / GDPR-K 合规):

数据最小化:
  - 不收集儿童真实姓名（使用随机生成的昵称）
  - 不收集精确位置（仅收集大致的服务器区域）
  - 照片 EXIF 数据默认清除
  - 不收集设备唯一标识符（使用会话ID）

数据隔离:
  - 儿童数据与成人数据物理隔离
  - 家长账号与儿童账号关联但数据分离
  - 不同儿童之间数据隔离（除非家长授权分享）

家长控制:
  - 家长 Dashboard: 查看/管理孩子所有数据
  - 数据导出: 家长可导出孩子所有数据
  - 数据删除: 家长可一键删除孩子所有数据（"被遗忘权"）
  - 权限管理: 家长控制每个功能的开关

通信限制:
  - 无自由文字聊天（预设短语 + 表情）
  - 无语音聊天
  - 无外部链接
  - 无广告
  - 无内购（或仅家长可操作）
```
